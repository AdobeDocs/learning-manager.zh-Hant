# 貢獻文章

我們歡迎來自我們社群以及來自文件團隊外的 Adobe 員工的貢獻。

## Adobe 開源行為準則

本專案已採用 [Adobe 開源行為](code-of-conduct.md) 準則或 [.NET 基金會行為](https://dotnetfoundation.org/code-of-conduct)準則。 欲了解更多資訊，請參閱貢獻[&#128279;](contributing.md)文章。

## 關於你對 Adobe 內容的貢獻

請參閱 [Adobe 文件貢獻者指南](https://experienceleague.adobe.com/docs/contributor/contributor-guide/introduction.html)。

你如何貢獻取決於你是誰以及你想要帶來的改變類型：

### 小幅變動

如果你是出於善意貢獻一些小更新，請造訪該文章，點擊 **文章中的編輯** 連結，該連結會指向該文章的 GitHub 來源。 然後，只要用 GitHub 介面來更新就好。 更多資訊請參閱一般 [Adobe Docs 貢獻者指南](https://experienceleague.adobe.com/docs/contributor/contributor-guide/introduction.html) 。

您在本資料庫中提交的文件與程式碼範例所提交的輕微修正或澄清，均受 Adobe 使用條款約束。

### 社群成員帶來重大變更或新文章

如果你是 Adobe 社群的一員，想要創建新文章或提交重大變更，請使用 Git 倉庫中的問題標籤提交問題，開始與文件團隊的對話。 一旦你同意了計畫，就需要與員工合作，透過公共與私人資料庫的工作，協助將新內容引入。

### Adobe 員工的重大變動

如果你是 Adobe Experience Cloud 解決方案產品團隊的技術寫手、專案經理或開發者，且你的工作是貢獻或撰寫技術文章，你應該使用私人倉庫。`https://git.corp.adobe.com/AdobeDocs`

## 工具與設置

社群貢獻者可以使用 GitHub 介面進行基本編輯，或分支倉庫以做出重大貢獻。

詳情請參閱 [Adobe 文件貢獻者指南](https://experienceleague.adobe.com/docs/contributor/contributor-guide/introduction.html) 。

## 如何使用 markdown 格式化你的主題

這個倉庫裡的所有文章都使用 GitHub 風格的 markdown。 如果你不熟悉 Markdown，請參考：

* [Markdown 基礎](https://help.github.com/articles/getting-started-with-writing-and-formatting-on-github/)
* [可列印的折扣速查單](https://guides.github.com/pdfs/markdown-cheatsheet-online.pdf)

## 唱片公司

在公開倉庫中，自動標籤會被分配給拉取請求，以協助我們管理拉取請求的工作流程，並讓你知道拉取請求的狀況：

* **變更已發送給作者**：作者已收到待處理的拉取請求通知。
* **準備合併**：我們的拉取請求審查團隊已準備好審核。
