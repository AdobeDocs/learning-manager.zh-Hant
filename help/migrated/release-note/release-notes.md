+++更新 103：Adobe Learning Manager 2025 年 11 月版本

發行日期：2025年11月25日

## 這次發布有什麼新意

### 體驗建構器

Experience Builder 現已適用於所有 Adobe Learning Manager 帳號，讓客戶能利用低程式碼元件與彈性版面打造完整品牌化的學習體驗。 Experience Builder 允許管理員建立完全行動響應式的客製化學習入口與頁面。

對於現有帳號，經驗建構器更新會自動推出。 經驗建構器功能不適用於仍使用舊版 UI 或經典版 UI 的帳號。

欲了解更多資訊，請參閱 [體驗建樑](/help/migrated/administrators/feature-summary/experience-builder/overview.md) 。

### Adobe Learning Manager 行動網頁介面

行動網頁介面已基於現代程式碼基礎重建，以提供更快的載入時間、提升的響應速度，以及跨裝置更直覺的學習體驗。

所有新帳號預設已啟用更新的行動網頁體驗。 現有帳號也會自動升級，除非啟用了社交學習。 在這些情況下，先前的行動體驗將持續運作，直到社會學習遷移到新的相容框架。

>[!NOTE]
>
>新的 Adobe Learning Manager 行動網頁介面不支援社交學習。

### 流體球員增強

過去，流體玩家的「下一」按鈕不會顯示即將推出的模組名稱，這讓學習者難以預測學習旅程的下一步。

透過這項改進，「下一」按鈕現在包含下一個模組的標題，讓學習者能夠：

* 在他們繼續之前，先了解接下來會有哪些內容。
* 以更清晰且自信的方式導航模組。
* 在整個課程流程中保持更好的上下文。

![](assets/fluidic-player-next.png)
_流體播放器現在會顯示下一個按鈕，上面有即將推出模組的標題_

### L1 反饋形式

L1 回饋只能在個別學習物件中設定，這限制了彈性，也讓管理員無法在目錄中使用不同的表單。 管理員現在可以建立並使用多個 L1 回饋表單，每個回饋表單都能分配給特定課程、學習路徑及認證。 該表單現在支援多種題型，包括課程效能、NPS、李克特量表及自由文本題。

回饋表單可以翻譯成其他語言，同時在所有地區保持相同的問題結構。 報告現在包含回饋表單名稱與回饋表單版本，以協助辨識每項回應的表單。

**使用案例**

* 為不同的培訓計畫分配量身訂做的回饋表單。 例如，銷售課程可以使用以NPS為主題的問題，而合規課程則可以使用課程效能與李克特量表問題來衡量清晰度與政策理解度。
* 根據學習模組使用不同的回饋表單。 例如，混合模組、自學模組和教室模組有三種不同形式。

欲了解更多資訊，請參閱 [L1回饋表](/help/migrated/administrators/feature-summary/l1-feedback-form.md) 單。

## 基於角色的電子郵件範本

### 客群

管理員與作者管理電子郵件範本

### 概觀

你（管理員或作者）可以根據收件人的角色、學習者、經理或講師來啟用或停用電子郵件範本。 它讓你能更好地掌控與學習者溝通的類型與頻率。

### 使用情境

* 管理員和作者需要在不通知管理者的情況下，透過電子郵件通知學習者（例如學習者註冊）。
* 有個選項可以針對特定角色批量啟用或停用電子郵件（例如，停用所有寄給經理的郵件）。

### 現有工作流程

如果目的是寄信給學習者但不寄給經理（例如課程報名），目前的範本會寄信給所有職務。

### 有什麼新鮮事

* 電子郵件範本介面上的基於角色的切換功能。
* 針對特定角色進行批量啟用或停用。
* 依範本可視化角色。

更多資訊請參閱本文[&#128279;](/help/migrated/administrators/feature-summary/email-templates.md#enable-or-disable-email-at-a-role-level)。

## 為已完成課程的學習者提供更佳的內容版本控制

### 概觀

當已完成課程的內容模組更新到新版本時，已完成課程的學習者仍會在背景接收更新後的模組版本。 這導致他們嘗試重溫賽道時出現意外的狀況，因為更新的內容並非為他們設計。

### 現有行為

當作者更新課程內容到新版本，並選擇只套用給尚未開始課程的學習者時，Adobe Learning Manager 仍會更新已完成課程的學習者版本。 因此，這些學習者無法如預期般存取原始內容。

此強化確保當課程模組更新並設定僅適用於尚未開始的學習者時，已完成課程的學習者仍可不間斷地存取原始版本。

### 改變了什麼

Adobe Learning Manager 現在為作者提供更清晰的內容更新管理選項。 作者可以更新課程中已有的內容。 新增版本時，版本號會顯示在內容旁。

當管理員造訪內容更新的課程時，他們會在新版本旁看到一個更新按鈕。 管理員也會看到清晰的更新選項，以選擇新內容版本如何套用給學習者。

| 學習者狀態 | 現在更新 | 更新 最終 | 更新尚未開始 |
|---|---|---|---|
| 未註冊 | V2 | V2 | V2 |
| 尚未開始 | V2 | V2 | V2 |
| 進行中 | V2 * | V1 → V2 * | V1 |
| 完成 | V2 * | V2 * | V1（保存） |

(*)表示模組在版本更新時會被重置。

在「更新未開始」時，已完成課程的學習者仍可繼續看到原始內容版本（V1）。 這避免了意外的回放問題，並確保學習者重溫已完成課程時能有一致的體驗。

### 內容更新選項

當管理員點擊 **[!UICONTROL Update]**&#x200B;時，他們可以從以下選項中選擇：

* **[!UICONTROL Update all Learners now]**：立即將內容更新套用於所有學習者。 未開始、進行中及已完成的學習者會立即切換到新版本。
* **[!UICONTROL Update all Learners eventually]**：將更新分階段應用給所有學習者。 未開始與完成的學習者現在會收到新版本。 進行中學習者完成當前版本後會收到更新。
* **[!UICONTROL Update only not started Learners]**：僅將更新應用於尚未開始課程的學習者。 進行中及已完成的學習者仍保留原始版本。

### 使用者介面變更

| 舊唱片公司 | 新選項 |
|---|---|
| 將內容更新應用給所有學習者 | 立即更新所有學習者:Apply 所有學習者的內容更新 |
| 申請更新給尚未開始的學習者 | 僅更新：僅未開始的學習者：適用於尚未開始的學習者 |
| 在學中學習者完成後會收到最新消息 | 最終更新所有學習者：最終為所有學習者應用內容更新 |

## 流體演奏者的改良

此版本為 Adobe Learning Manager 帶來全新流體播放器，外觀現代化，學習體驗更直覺。 更新後的播放器介面更乾淨、導航控制更精良，以及桌面與行動裝置間統一的設計。 學習者可以輕鬆存取並互動任何螢幕模式，包括直向、橫向、全螢幕或標準視圖。

* **全新流暢的玩家皮膚**：現代化外觀，UI元素更新。
* **優化的導航控制**：更流暢且直覺的導航，提升學習體驗。
* **優化設計**：在桌面與行動裝置間提升外觀與可用性，支援直向、橫向、全螢幕及標準螢幕模式。

所有透過 Adobe Learning Manager 存取內容的學習者，不論使用 Native 或 Headless 體驗，都會自動看到這些新的介面與導航改進。 學習者可能會注意到，因為版面與操作的更新，與玩家互動的方式有些微改變。

>[!NOTE]
>
>此增強功能僅適用於靜態內容。

有關流體演奏者的更多資訊，請參閱本文[&#128279;](/help/migrated/learners/feature-summary/fluidic-player.md)。

## 透過 SAML 建立介面語言

Adobe Learning Manager （ALM） 現在接受語言的 SAML 屬性。 此屬性隨後會映射到使用者的介面與內容語言設定，確保以偏好語言與 LMS 順暢互動。 這些語言設定的配置透過身份與存取管理（IAM）平台管理，採用 SAML 進行單點登入（SSO）。 這支援由服務提供者（SP）發起的登入，也支援身份提供者（IdP）發起的登入，讓使用者能以自己選擇的語言查看介面與內容。

欲了解更多資訊，請參閱本文[&#128279;](/help/migrated/administrators/feature-summary/set-up-interface-language-through-saml.md)。

## 遷移 API 的強化

過去，透過 API`GET /bulkimport/cansync` 遷移的帶有外部連結的活動模組（和 `POST /bulkimport/startrun`）在存取連結後不會顯示給學習者這個 **[!UICONTROL Mark as Complete]** 選項。 這個問題已經解決了。 現在，透過 API 遷移外部連結的活動模組，學習者會正確顯示該 **[!UICONTROL Mark as Complete]** 選項。

## Learner 應用程式中的排序功能

學習者應用程式的排序功能會根據內容和介面語言提供個人化課程推薦。 此改進簡化了學習者尋找偏好語言課程及更智慧排序選項的過程。

欲了解更多資訊，請參閱本文[&#128279;](/help/migrated/learners/feature-summary/catalogs.md#sorting-functionality-in-the-learner-app)。

+++更新 89：2023 年 3 月 Adobe Learning Manager 發布

**上映日期：** 2023年4月1日

### 這次更新有什麼新意

**教官主導訓練（ILT）體驗的提升**

教官主導訓練（ILT）體驗進行了多項提升。 主要增強功能包括——能依地點篩選教室課程、可切換實例（VILT）而不失去進度、新增「排程助理」以管理教師與教室預約衝突、能將「技能」附加給教師並依技能選擇教師的能力。

**觀察檢查清單**&#x200B;的改進：

作者現在可以選擇「經理」和「商店經理」作為檢查清單的觀察者。 經理可在管理員介面中查看並完成檢查清單，無需切換角色為講師。 當檢查清單被分配給經理時，通知會發送給他/她。

**使用任何應用程式或智慧型手機相機掃描學習管理員的 QR 碼**

學習者現在可以使用任何 QR 碼掃描應用程式或智慧型手機相機，掃描學習管理員產生的 QR 碼，以完成課程報名、完成等操作。

**報告強化**

新的講師使用報告、訓練回顧報告、工作輔助報告及其他報告改進。

**支援「混合」會議**

Adobe Learning Manager 現在支援建立「混合」講師主導訓練（ILT）課程的功能。 虛擬 ILT 課程可透過選擇性地點資訊建立，若學習者在場可選，也能親自參加課程。

**更佳的課堂與虛擬學習學習進度追蹤**

教室與虛擬 ILT 模組現在提供報告學習者測驗狀態（通過或不通過）及出席狀況的功能。 因此，出席率與小考成功率均可作為決定學習進度的考量。

**Microsoft Teams 的 Adobe Learning Manager 應用程式**

Microsoft Teams 上全新的 Adobe Learning Manager 應用程式旨在促進工作流程中的學習並促進社交學習。 學習者將能在 Microsoft Teams 平台內存取學習內容，無需切換到瀏覽器。 請聯絡您的 CSAM，了解 MS Teams 上 Adobe Learning Manager 應用程式的測試版。
