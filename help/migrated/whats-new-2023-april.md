# 本次發布（2023年4月）有什麼新內容

## Microsoft Teams 的 Adobe Learning Manager 應用程式

Microsoft Teams 上全新的 Adobe Learning Manager 應用程式旨在促進工作流程中的學習並促進社交學習。 學習者將能在 Microsoft Teams 平台內存取學習內容，無需切換到瀏覽器。 請聯絡您的 CSAM，了解 MS Teams 上 Adobe Learning Manager 應用程式的測試版。

欲了解更多資訊，請參閱 [Microsoft Teams](/help/migrated/adobe-learning-manager-app-microsoft-teams.md) 的 Adobe Learning Manager 應用程式。

## 教官主導訓練（ILT）體驗的提升

教官主導訓練（ILT）體驗進行了多項改進。 主要改進包括：依地點篩選課程、可切換實例（VILT）而不損失進度、新增「排程助理」以管理預約課程衝突、能將「技能」附加給講師，並依技能選擇講師。

### 改變了什麼

* 管理員應用程式的技能頁面有上傳技能-講師對應的選項。
* 在技能頁面上，新增了一個專欄「講師」。 該欄顯示該技能對比的教練人數。 如果你點擊「講師」欄位中的數字，會跳出一個視窗，顯示分配給某項技能的講師人數。

欲了解更多資訊，請參閱 [「指派技能給講師](/help/migrated/administrators/feature-summary/skills-levels.md#assign-skills-to-instructors)」。

![講師 CSV](assets/instructor-csv-new.png)

### 排程助理

處理安排教師與教室或虛擬教室的衝突。 如果你想知道任何講師在分配課程前的時間和日期，請使用排程助理。

作者、管理員及自訂管理員/作者皆可使用排程助理。

欲了解更多資訊，請參閱 [排程助理](/help/migrated/authors/feature-summary/courses.md#scheduling-assistant)。

## 全新 Adobe Learning Manager 行動應用程式

一款全新的 Adobe Learning Manager 行動應用程式，適用於 Android 和 iOS，讓學習者隨時隨地都能順利學習。 該應用程式讓學習者能在行動裝置上接受訓練，之後再在桌面端繼續進行。 支援通知、檔案上傳、輕鬆書籤與分享學習內容、社交學習等功能，為學習者提供更多彈性，並支援即時學習。

![新的行動地圖介面影像](assets/mobile-app.png)

請從 Google Play 商店和 Apple App Store 下載手機。 舊版應用程式的使用者將被提示下載並安裝新應用程式。

在此版本中，應用程式支援以下功能：

新手機應用程式在此版本中支援以下關鍵功能：

* 強化搜尋，包括在搜尋輸入框中顯示學習者近期搜尋及組織內熱門搜尋。
* 能夠下載學習者成績單。
* 即興通知中心 - 在應用程式內以通知形式向學習者提供重要更新
* 能夠在目錄/我的學習頁面中排序和篩選學習物件
* 學習物件（LO）書籤功能 - 書籤的學習對象可在學習者首頁的「由我保存」區塊中提供。
* 強化版 LP 的支援
* 能夠將行動介面語言更改為任何 Adobe Learning Manager 支援的語言。
* 支援外部認證、CR/VC 會議及活動模組中的檔案上傳。

## 使用任何掃描 QR 碼的應用程式來掃描 Learning Manager 的 QR 碼

Adobe Learning Manager 現在支援一種更直覺的掃描方式，利用原生相機應用程式，無需下載掃描 QR 碼的應用程式。

為了支援先前的工作流程，即可透過 ALM 應用程式選單掃描 QR 碼，本版本中我們提供了如何切換到新工作流程的資訊。

隨著 QR 碼工作流程的改進，先前產生的舊 QR 碼將不再支援。 因此，如果你用舊版 Learning Manager 為本版本後的教室或虛擬化工作產生 QR 碼，那麼你必須產生新的 QR 碼。

### 推薦公告增強功能

在本版本中，我們將推薦條從組織更新為基於使用者群組的推薦，該使用者群體是學習者所屬，以顯示學習者相關的課程。

管理員可根據學習者的個人資料，針對特定課程或培訓對象。 學習者可以根據所屬的個人檔案搜尋課程。

**改變了什麼**

公告頁面有一個選項，會告訴學習者為何會推薦此課程或訓練。

![建立公告](assets/create-announcement.png)

## 觀測檢查清單的改進

作者現在可以選擇經理和門市/地點經理作為檢查清單的審查員。 經理及門市/地點經理若被選為審查員，也能查看並完成團隊的檢查清單，而無需切換到講師角色。 講師持續檢視檢查清單。 一份新的「審查清單」通知會發送給註冊檢查清單實例的審查員（講師/經理）。

如果在檢查清單模組中新增經理為審查員，他們就能在管理員應用程式中檢視檢查清單。 講師應依預期持續檢視檢查清單。

## 其他強化

### 學習者搜尋

搜尋結果也依以下分類：

* Recent searches by you
* 貴組織中熱門搜尋

搜尋功能也為學習者提供了改進。 學習者現在可以使用雙引號「...」加上「+」和減「-」運算符，更快找到相關結果，並享有類似 Google 搜尋的體驗。

* 使用雙引號（「...」）搜尋包含該片語或單字的課程。 例如，輸入「資料科學」會回傳以「資料科學」開頭的課程。
* 使用+運算子確保只顯示包含特定片語或單字的結果。 例如，Computer Programming + Python 只會顯示包含「python」這個詞的電腦程式設計課程。
* 使用-操作符確保只顯示那些不含特定片語或單字的結果。 例如，Computer Programming -python 會顯示除包含「python」一詞外的所有電腦程式設計課程。

### 學習物件的衝突彈出視窗

當學習者有衝突的課程時，概覽頁面會顯示報名的彈出視窗。 如果從目錄頁面或總覽頁面註冊，且學習者已經有衝突的課程，則會觸發衝突的彈窗，包含衝突的課程細節。 即使學習者有衝突的課程，他們仍可註冊新的課程。

![警告](assets/learner-pop-up.png)

請注意，這則訊息只是警告。 你仍然可以報名修課。

### 新範本

我們新增了一個名為「因取消報名而取消的會次」電子郵件範本。 當使用者被取消註冊訓練時，會收到取消郵件。

### 課程新定價功能

管理員現在可以在管理員設定>基本資訊區塊中設定帳戶的貨幣。 管理員可以指定符號以及 ISO 貨幣代碼，例如美元、英鎊等。 所有新帳號的預設值都是 $。 此變更僅適用於非學習型應用程式。 管理員在預覽課程時可以看到這些變化。 貨幣符號也出現在學員成績單與訓練報告中。

欲了解更多資訊，請參閱 [管理員設定](/help/migrated/administrators/feature-summary/settings.md)。

### 交換實例

註冊在特定課程實例的學習者可以查看該課程所有可用實例的清單，並切換到更適合自己的實例。 切換的原因可能是學習者錯過了之前的實例，或是新實例的課程時間更合適，或是其他原因。

學習者在課程中取得的任何進展、學習者測驗分數等，都會帶入新的實例。 此功能主要針對教室與混合課程設計，但我們也會支援各類課程，包括自學進度的課程。

「檢視所有實例」這個選項不論實例切換設定如何都會出現。 學習者現在可以看到一個選項，可以查看所有實例，若同一課程有多個實例。

若啟用實例切換選項，學習者可切換至多個實例直到完成課程。 若該選項被停用，學習者可查看實例，但無法註冊課程。

當學習者選擇 **「檢視所有實例**」時，就能看到所有課程實例。

自訂管理員和作者若有必要的權限，也可以在實例層級查看此選項。

更多資訊請參閱 [公告](/help/migrated/administrators/feature-summary/announcements.md#as-email)。

### 新的電子郵件範本

在本版本中，我們新增了給跳級主管的電子郵件範本。 這些範本是為了讓經理收到關於第二級報告缺少任何課程、學習路徑或認證截止日期的電子郵件時使用。 此外，當他們的第二級報告接近完成這些課程、學習路徑或證照的截止日期時，他們也會收到電子郵件。

* 跳過課程完成的升級階段 - 截止日前
* 錯過課程截止日的跳級升級
* 跳過層級升級以完成學習路徑 - 截止日前
* 跳過因錯過學習路徑截止日期而進行的升級
* 完成認證的跳過升級階段 - 截止日前
* 錯過認證截止日的升級

這些選項預設是啟用的。

## 本版本的 API 變更

### 新報告

我們在 jobs API 中新增了一個屬性 jobType。 屬性接受以下值：

* **生成教師利用報告**：回傳教師的使用報告。
* **生成JobAidMetadataReport**：回傳工作援助報告的元資料。

**Endpoint**： POST /primeapi/v2/jobs

生成JobAidMetadataReport 請求：

```javascript {line-numbers="true"}
{ 
    "data": { 
        "type": "job", 
            "attributes": { 
                "description": "description of your choice", 
                "jobType": "generateJobAidMetadataReport" 
            } 
    }
} 
```

生成JobAidMetadataReport 回應：


### 維多利亞的教室

執行基於地點的VC課程搜尋。 資源模型中新增了 roomLocation 屬性，用以表示建立 VC 課程時提供的自由位置。

我們做了以下調整：

**LO**

新增查詢參數 filter.loFormat=Virtual Classroom，用於 learningObjects API。

**端點**：取得 /primeapi/v2/learningObjects

**賽程表**

新增一個查詢參數，filter.allSessions=false，用於行事曆 API。 參數的預設值為 false。 如果為真，則 API 會回傳學習者的所有行事曆會話。

**終點**： `GET /primeapi/v2/users/<id>/calendar?filter.allSessions=false`

### 學習者搜尋歷史

**搜尋**

新增查詢參數 persistSearchHistory。 預設值為 true，該值會持續執行搜尋建議的查詢。

**端點**：GET /primeapi/v2/search？persistSearchHistory=true

**建議**

新增一個查詢參數，建議類型。 接受的數值如下：

* learnerHistory（預設）
* 帳目歷史

**端點**：GET /primeapi/v2/search/suggestions/？suggestionType=learnerHistory

### 使用者群組篩選

學習物件與搜尋 API 會提供篩選器，透過指定的篩選器擷取屬於使用者群組的所有學習物件。 API 支援以逗號分隔的篩選列表。

我們提供了一個新的篩選器 filter.lo.announcementGroups，用來擷取所有屬於使用者群組篩選器的 LO。

這可以是一個多值逗號分隔的清單以增加彈性，處理則基於多組間的「OR」操作。 也就是說，取得所有屬於該使用者群組的 LO。

### 學習者應用程式中 LOS 的公告用戶群組過濾

* GET /users/{userId}/userGroups API 新增了一個參數 filter.announcedGroupsOnly，該參數會取一個布林值（true/false）。 這只會過濾管理員公告的使用者群組。 此參數的預設值為假。
* GET /learningObjects API 新增了一個參數 filter.announcedGroups，可接受公告群組 ID 來篩選結果。
* GET /search API 新增了一個參數 filter.announcedGroups，可接受公告群組 ID 來篩選結果。
