# 新功能摘要 {#new-features-summary}

了解 2024 年 3 月版本 Adobe Learning Manager 中的新功能與增強功能。

探索一些最新的 Adobe Learning Manager 功能，例如：

1. 從外部來源匯入技能
1. 多重品牌支援
1. 檢查清單重新評估活動模組
1. 白標行動學習應用程式

>[!NOTE]
>
>請觀看這場 [線上研討會](https://learningmanager.adobe.com/app/learner?accountId=98632#/course/9212121) ，了解更多本版本的新功能。


## 這次發布有什麼新意 {#whatsnewandchanged}

### 從外部來源匯入技能

透過使用相關連結器，從內容提供者如 LinkedIn 和 Go1 匯入技能。 這項提升是 Learning Manager 能夠整合外部技能雲與人才管理系統的目標之一。 匯入的技能會加入學習管理員定義的技能，並在課程建立流程中提供給作者使用。 平台的技能搜尋功能也有所提升，以提供更佳的搜尋體驗，當帳號擁有大量技能時。

查看 [匯入技能](administrators/feature-summary/import-skills-external-sources.md) 以了解更多資訊。

### 客製化品牌

你現在可以根據帳號中可用的使用者群組自訂某些使用者介面元素——組織名稱、標誌和 UI 主題。 例如，擁有多個部門的組織可以為每個部門設定自訂標誌和 UI 主題。

>[!NOTE]
>
>此多重品牌功能不適用於管理員視圖。 他們在帳戶裡總會看到組織層級的品牌。 這是因為這是面向學習者的功能，管理員可能不想讓它加入他們的帳號。

欲了解更多資訊，請參閱 [多個自訂品牌](administrators/feature-summary/themes.md#multiple-branding) 。


## 針對擁有大量用戶群的帳號的變更

### 行政-課程或學習路徑頁面

例如，若課程註冊人數超過50,000人，則不會顯示學習者名單。 你可以在 *搜尋學習者* 搜尋列中搜尋學習者，或選擇 **搜尋欄頂端的下載** 連結下載學習者名單。

### 管理員-學習者頁面

搜尋任何使用者時，下載 **學習器** 與 **匯出** 選項會下載相同的報告。 同時，在搜尋使用者群組時，你現在可以下載該使用者群組中篩選過的使用者。 搜尋使用者群組時，**下載學習者名單**&#x200B;的變更為&#x200B;**為使用者群組**&#x200B;下載學習者清單。**匯出**&#x200B;選項再次下載整個學習者清單。

### 管理員-使用者頁面

#### 內部使用者

例如，若用戶數超過 50,000 人，系統會提示下載資料以便日後進行更詳細分析。 搜尋欄現在變得顯眼，並以以下格式 *顯示使用者，格式為 姓名、電子郵件 |UUID*。

>[!NOTE]
>
>UUID 只有在帳號啟用時才會顯示。

#### 外部使用者

對外部使用者來說，行為同樣適用。 若使用者數量眾多，您可以下載使用者資料，並透過搜尋 *格式取得使用者資料，格式為 Name， email |UUID*。

#### 使用者清理頁面

在用戶清理頁面，針對已刪除使用者，我們移除了「 **刪除**&#x200B;日期」的排序功能。 你只能在 UUID 上排序。

### 管理員-實例頁面

#### 課程或學習路徑

若註冊人數龐大，Adobe Learning Manager 將不會顯示學習人數。 取而代之的是，會有一個圖示，你可以選擇查看學習者人數，並導向學習者頁面。

學習人數將以近似值顯示。 例如，若學習人數超過 50,000 人，數值會顯示為 50K+。

### 管理員 - L1/L3 頁面

在L1回饋頁面，若課程報名人數眾多，學習者名單不會顯示。 你可以匯出使用者名單，方便日後進行更詳細的分析。

搜尋支援預先輸入，結果僅限於所選實例。

#### 出席人數與得分頁面

在頁面上，當你搜尋使用者時，搜尋會跨所有可用實例執行。 然而，結果是針對所選實例。

在出席頁面，如果你搜尋某個使用者群組，且該群組的使用者數超過10,000，不論是否註冊，你只能執行批量操作。 您將無法查看使用者名單。

如果使用者群組的使用者數量少於 10,000 人，則你可以同時執行個別使用者層級的操作以及批次層級的行動。 在這種情況下，使用者列表並未被停用。

### 管理員-認證頁面

在目前版本的 Adobe Learning Manager 中，如果有大量使用者註冊認證，你無法查看未註冊的學習者，因為&#x200B;**&#x200B;**&#x200B;狀態下拉選單是被停用的。

在本版本的 Adobe Learning Manager 中，如果註冊用戶數量較多， **狀態** 下拉選單只會顯示兩個選項—— **已** 註冊與 **未註冊**。 **預設選項為已**&#x200B;註冊。如果你選擇 **未**&#x200B;註冊，會顯示未註冊學習者名單。

#### 使用者群組變更

若使用者群組的使用者數量少於例如 50,000 人，狀態下拉選單會&#x200B;**&#x200B;**&#x200B;顯示所有選項——認證、指派及過期。

如果使用者群組中的使用者數量較多， **狀態** 下拉選單只會顯示兩個選項—— **根據新設計，已註冊** 與 **未註冊**。

## 進階搜尋能力

在本版本中，我們提升了搜尋體驗。 搜尋結果不僅根據元資料擷取，還結合語意及內容內搜尋，以根據精確度、近期性及相關內容推導結果。

此變更反映了以下內容：

* 目錄與我的學習頁面：課程、學習路徑和認證上的懸停動作已被移除。
* 搜尋欄的出現。
* 在學習應用程式中新增了篩選標籤。

欲啟用搜尋功能，請聯絡 Adobe Learning Manager 的 CSAM 團隊。

## 使用者介面變更 {#ui-changes}

### 課程創建頁面

在將賽道對應到技能等級時，技能列表會優先搜尋。 換句話說，就是搜尋技能，你會看到一份與搜尋詞相符的技能清單。

### 使用者群組

#### 管理員：學習者頁面

搜尋任何使用者時，下載 **學習器** 與 **匯出** 選項會下載相同的報告。 同時，在搜尋使用者群組時，你現在可以下載該使用者群組中篩選過的使用者。 搜尋使用者群組時，**下載學習者名單**&#x200B;會切換為&#x200B;**下載使用者群組**&#x200B;**的學習者清單。匯出**&#x200B;選項再次下載整個清單。

## 報告變更

* 訓練報告中的標籤與技能欄位改為標籤與技能。
* 新增了遊戲化審計追蹤[&#128279;](administrators/feature-summary/reports.md#gamification-audit-trail)報告。
* 若帳號包含超過 280,000 名被分配到某項技能的學習者，則技能學習者報告會以壓縮 CSV 格式下載。如果帳號學習者少於 250,000 人，該報告會被下載為 CSV。在管理員頁面，選擇&#x200B;**管理員**>**技能**> **&#x200B;**&#x200B;技能>**學習者**。報告會以 CSV 格式下載。
* [會議摘要報告](administrators/feature-summary/reports.md#session-summary-report)新增兩個欄位——地點資訊與地點區域。

## 教室設置的變革

根據 [管理員設定](administrators/feature-summary/classroom.md#classroom-settings)，作為作者， [你可以建立、修改和刪除位置](administrators/feature-summary/classroom.md#add-classroom-location)。

>[!NOTE]
>
>在新增地點與目錄標籤時，作者（在課程建立頁面）和管理員（在實例頁面）會分別看到自動填入的地點與目錄標籤清單。

作為管理員，你可以對作者修改或刪除教室位置施加限制。 欲了解更多資訊，請參閱 [教室設定](administrators/feature-summary/classroom.md#classroom-settings) 。

## 彈性學習路徑的變更

所有新舊帳號都會在學習者應用程式中開始包含「註冊截止日」、「退選截止日期」及「名額限制」，以提供彈性學習路徑。學習者現在可以註冊彈性學習路徑，而無需選擇任何課程實例。

## 學習計畫的新觸發器

學習計畫設定頁面新增了一個觸發器。 作者與管理員現在能在學習者未通過課程模組時觸發行動。

欲了解更多資訊，請參閱 [學習計畫](administrators/feature-summary/learning-plans.md) 。

## 新提交狀態

作為講師，您現在可以依據狀態、結束審查、待提交、通過及未通過來篩選學習者的檔案提交。

查看 [投稿狀態](instructors/feature-summary/learners.md#filter-file-submissions) 以獲取更多資訊。

## 檢查清單的強化

在 2024 年 3 月的 Adobe Learning Manager 版本中，檢查清單工作流程的改進如下：

### 未完成清單後禁止進度

在建立檢查清單時，作者可以在強制檢查清單區塊中選擇 **啟用** 。 若未通過檢查表，學習者將無法繼續該模組。 他們只有通過檢查清單才能繼續。

檢查清單的審查員，即講師或管理者，可以檢查清單的狀態。 審查員也可能將學習者的檢查清單順序打亂。

### 檢查清單的重新評估

在建立檢查清單時，作者可在重新評估區塊選擇 **啟用** 。 這樣做讓管理者或講師能夠重新評估學習者，直到他們通過檢查清單。

若模組為必修，則預設會勾選重新評估的勾選框。

當重新評估啟用時，教師或經理也可以將檢查清單狀態從未通過更改為通過。

在清單頁面，講師可以看到待處理狀態的學習人數。 作為講師，你可以評估學習者並決定他們通過或不及格。 若學習者處於失敗狀態，只有在未啟用重新評估時才能查看檢查清單。

這表示 **在建立檢查清單時，重新評估區塊中沒有選擇啟用** 勾選框。 如果勾選此勾選框，即可在講師清單頁面看到「檢視/重新評估」按鈕。

點擊按鈕後，你可以重新評估學習者，並標記他們通過或不及格。

>[!NOTE]
>
>這兩個功能——重新評估和將檢查清單設為強制——只適用於新建立的模組。 課程一旦發布，這些功能就無法再開關。


查看 [建立清單](authors/feature-summary/courses.md#checklist-fail) 以獲取更多資訊。

## 其他強化

### 會話相關電子郵件通知

在早期的 Adobe Learning Manager 版本中，學習者不會發送與工作階段相關的電子郵件、會議細節已更新、會議邀請及課程提醒，當：

* 學習者已完成課程，
* 課程中新增課程，或
* 現有的會議內容有所變動。

在 2024 年 3 月發布的 Adobe Learning Manager 中，以下是新的變更：

* 課程詳情已更新及課程邀請（針對學習者與講師）
   * 未來課程中，課程詳情更新&#x200B;**、**&#x200B;已註冊學習者及現有講師的課程邀請&#x200B;**電子郵件**&#x200B;將被取消。過去的課程，已更新&#x200B;**的課程詳情及**&#x200B;已註冊學習者與現任講師的課程邀請&#x200B;**電郵**&#x200B;將維持原狀。
* 提醒郵件（給管理員與學習者）
   * 未來的課程將僅 **發送會議提醒** 郵件。

>[!NOTE]
>
>郵件不依課程完成時間而定。


### AEM 參考網站變更

在 AEM 參考網站中，我們新增了將管理員刷新令牌加入學習者存取令牌的支援。

### 隱藏老師提交的作品

學習者透過檔案提交工作流程上傳檔案後，若講師未對提交採取任何行動（批准或拒絕），提交網址會在預設天數後隱藏於檢視中。 請聯絡 Adobe Learning Manager 的 CSAM 團隊，設定或更改天數。

### 產品術語變更

我們已將「實例&#x200B;*」和*「學習者&#x200B;*」欄位*&#x200B;加入產品術語詞彙。

### 行動應用程式變更

## 本版本的 API 變更

### 學習器 API

在本版本中，我們新增了 API 支援，讓學習者能在使用者群組層級查看品牌標誌與個人化主題。

API：/account 和 /user？include=account 回傳四個欄位，這些欄位會被覆蓋，這些欄位會被覆蓋，該欄位屬於屬於 logoUrl、logoStyling 和 themeData。

### 新屬性

learningObjectResource 新增了一個屬性 isExpiredSubmission，用來顯示該資源中的提交是否已過期。

* GET /account API：回傳新屬性 **expireSubmissionDuration** X，其中 X 是設定的天數。 若未設定，則返回 0
* GET /LO API 與資源包含新屬性 **isExpiredSubmission**」，真或假。
   * 沒錯，如果投稿已過期且沒有顯示「submissionUrl」。
   * 若為 False，則該提交尚未過期，並會取得「submissionUrl」。

### 檢查清單中的 API 變更

一門課程可以包含多個模組，其中 Checklist 是一種模組。 此檢查清單模組由講師評估，並可依評估結果評為失敗或成功。

但兩種情況下，檢查清單狀態都會標示為已完成，這樣課程就會被標記為完成。

在此版本中，LO API 包含參數 *isChecklistMandatory*。 若值為真，則該檢查清單為強制性。

### 支援多地區

管理員現在可以下載他們選擇語言的 L1 回饋報告。 不過，你目前還無法下載 Power BI 的 L1 回饋報告。 在 API 請求中，使用參數 preferredLocale 指定你選擇的區域。

### 實例摘要計數的變動

此規定適用於課堂/VC課程註冊人數超過1000人的帳戶。

若數值低於 1000，註冊會使快取失效，並回傳更新後的 GET 摘要 API 值，如註冊數、完成數及席額限制。

若帳號啟用此功能且註冊數超過 1000，則會從快取中取回數值。

### 已淘汰路徑

目前，Learning Manager API 採用圖形資料結構，允許你透過 include 遍歷 API 模型來取得資料。 即使你可以遍歷一個 API 最多七層，但用單一 API 呼叫擷取資料仍然計算成本高昂。

我們建議所有現有及新客戶多次撥打小額電話，而非一通大型電話。 此方法可防止通話中載入不受歡迎的資料。

### 登入存取與使用者稽核報告 工作 API 的檔案變更

本版本將保留登入存取報告長達五季，使用者稽核報告則保留六個月。 若您想下載此期間之前的資料，必須通過存檔參數，指定季度與年份。 請參考樣品有效載荷。

```
{
    "data": {
        "type": "job",
        "attributes": {
            "description": "description of your choice",
            "jobType": "generateLoginAccessReport",
            "payload": {
                "fromDate": "2023-04-01T18:30:00.000Z",
                "toDate": "2023-04-30T18:30:00.000Z",
                "archive": {
                    "quarter": "4",
                    "year": "2021"
                }
            }
        }
    }
}
```

如果你嘗試下載 **超過五個季度的登入存取** 報告，會顯示錯誤訊息。 如果你嘗試下載 **超過六個月的使用者稽核** 報告，也會顯示類似的錯誤訊息。

## 本次更新修正的錯誤 {#bug-fixes}

* 當學習者註冊一門課程後嘗試註冊另一門課程時，會顯示警告訊息。
* 使用者群組即使被刪除，搜尋中仍可見。
* 當使用者觸發大量資料的學習者成績單時，學習者成績單佇列會被阻塞，阻止新的請求。
* 若子帳號請求其父帳號分享報告，父帳號無法執行。
* 課程和學習路徑的網址會被導向到錯誤的位置。
* 學習者在目錄頁面點擊課程連結時，會間歇性地查看另一門課程的實例。
* **第一次註冊後，取消登記**&#x200B;按鈕不會如預期顯示，但重新整理後按鈕會顯示。
* 你無法儲存內容或測驗名稱中空格的檔案。
* 在經理核准的課程中，你可以重新註冊學習者到使用者群組。
* 在某些情況下，若嘗試新增一個啟用欄位，會出現錯誤訊息「無法儲存活躍欄位。」的錯誤訊息。
* 相關課程區塊的課程卡中，該文字會溢出於課程名稱。
* 切換實例並註冊學習者後，舊實例仍存在於 Outlook 行事曆中。
* 當同儕帳號的學習者嘗試選擇課程縮圖時，會跳出錯誤訊息。
* 當學習者註冊課程時，會收到多次報名通知。
* 如果使用者手動更改連接器中建立的目錄名稱，就會產生新的目錄，課程則會被發佈到錯誤的目錄中。
* 非活躍帳號的用戶仍會收到訂閱郵件。

### API 相關錯誤修正

* API GET/users 不會取得管理器的詳細資訊。
* 在一個帳號中，使用者是透過排程的 FTP 使用者匯入，在排定的停機期間建立的。
* 在行動應用程式或沉浸式模式中，刪除或退休課程實例後，選擇下一個活躍實例後， **顯示的是「註冊興趣** 」按鈕，而非 **「註冊**」。
* 當同儕帳號的學習者嘗試使用 Learning Object API 選擇課程縮圖時，會顯示錯誤 403 禁止。
