# 常見問題（報告建置器）

**範本和報告有什麼不同？**

範本是由 Adobe Learning Manager 預先建構好的報表配置。 它們設計用於常見使用情境，隨時可立即下載，且為唯讀。 你無法編輯它們。 報告是你自己儲存的設定。 你可以從零開始製作，或是複製範本並編輯副本來建立它們。 報告會出現在你的 **報告** 標籤;範本則出現在 **範本** 標籤。

**我可以直接編輯範本嗎？**

不。 範本是唯讀的。 若要自訂範本，請選擇 **複製** 以建立可編輯的副本。 你的變更會以新報告形式儲存在 **報告** 標籤中，不會影響原始範本。

當資料中的欄位與主記錄一對多關係時，會出現重複列。 常見原因包括多位講師的課程（每位講師每堂課一列）以及多位作者的學習物件。 為了解決這個問題，可以將包含多個值的欄位（例如 **講師名稱** 或 **作者名稱**）作為欄位。

**為什麼日期會顯示在UTC中？**

報告建構器在本版本中回傳UTC的日期值。 你帳號設定的時區會在未來版本中套用到日期欄位。 分析基於日期的資料時，請考慮與你帳戶主要時區相關的 UTC 偏移。

**新的註冊或完成資料通常需要多久才會出現？**

報告建構器從 Adobe Learning Manager 的資料庫中擷取資料，該資料庫的最大延遲約為從系統事件發生起約 15 分鐘。 如果你剛記錄了註冊或完成，但報告中沒有顯示，請至少等15分鐘再下載一次。

**報告中有列數或欄位的限制嗎？**

報告數量限制約為 100 萬行。 本版本對欄位數量沒有限制。 如果你的報告需要超過 100 萬筆資料列，請使用篩選器縮小範圍。

**從 Report Builder 匯出報告時，有檔案大小限制嗎？**

目前，報告建構器不支援超過 5GB 的匯出報告檔案。 如果你的報告預期會超過這個大小，可以考慮加裝額外篩選器或減少資料列數，讓匯出量控制在 5GB 以下。

**我可以透過 API 或自動化方式拉取報告建構器的資料嗎？**

未來計畫推出自動化 API 存取報告建構器報告的功能。 目前版本中，報告可透過報告建置器介面手動下載，或透過訂閱功能排程接收。
