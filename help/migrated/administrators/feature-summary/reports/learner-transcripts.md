# Adobe Learning Manager 中的學習者成績單

## 概觀

Adobe Learning Manager （ALM） 中的學習者成績單（Learner Transcript）允許管理員在課程、模組、學習路徑及認證中細緻追蹤學習者的進度。 成績單資料有助於績效審查、合規追蹤、稽核及外部報告需求。

>[!NOTE]
>
>學習者成績單可供管理員、自訂管理員、經理或學習者下載。

學習者逐字稿下載體驗及最終檔案會依使用者角色而異。 管理員與自訂管理員可為多位學習者產生成績單，並可存取更廣泛的資料集，而學習者只能透過個人檔案設定下載自己的成績單。 下載介面也有所不同：管理員使用報告區，而學習者則從個人檔案存取成績單。 下載的檔案可能會根據角色和權限，包含不同的欄位和細節層級。

對於學習者而言，他們必須啟動個人檔案設定，然後下載學習成績單的 Excel 檔案。 這份為個別學習者製作的逐字稿，詳述其個人學習歷程。 它包含學習路徑、課程、實例和模組的名稱，以及像是報名、完成和截止日期等重要日期。 它還會追蹤學生的進度，包括狀態、成績、測驗分數（包括最高分和最高分）以及嘗試次數。 此外，還會顯示訓練編號、時長、退役日期、價格以及任何提交留言。 本報告提供單一學習者的參與度與表現的全面概述。

組織可利用學習者逐字稿將學習行為資料加入企業資料倉儲，並可將學習資料與其他企業資料結合，分析學習行為與其他流程資料之間的相關性。

## 目的與效益

* 建立符合法規要求的審計紀錄，附有時間戳記及完成證明。
* 將學習活動與員工發展及技能獲取連結起來。
* 追蹤需要強制訓練職位的認證狀態。
* 支援學習計畫成效的量化衡量。

## 學習者成績單的使用案例

Adobe Learning Manager 中的學習者成績單追蹤訓練、合規與技能發展，使各部門能驗證完成度並評估整個組織的計畫成效。  以下使用案例說明學習者成績單如何支援組織在合規、技能追蹤及計畫效能方面的需求。

* 金融服務組織必須提供所有面向客戶的員工在法規截止日前完成強制合規訓練的證明。
* IT 部門需要評估現有的 Java 程式設計能力與未來專案需求。
* 人資希望評估新員工入職計畫在不同部門的成效。
* 營運團隊需確保所有現場技術人員均持有必要的安全認證。

## 存取控制與權限

**管理者**

* 管理員可以為所有學員在所有目錄中產生成績單。
* 自訂管理員只能查看其分配的使用者群組與目錄中學習者的成績單。

**範圍限制**

* 自訂管理員只能查看其分配使用者群組內學習者的成績單。
* 報告只會包含管理員有權限存取的目錄中的學習物件。

## 如何產生學習者成績單

1. 以管理員身份登入 Adobe Learning Manager。
2. 從左側導覽選單選擇 **[!UICONTROL Reports]** 。
3. 在報告中選擇 **[!UICONTROL Custom Reports]** ，然後選擇 **[!UICONTROL Excel Reports]**。
4. 選擇 **[!UICONTROL Learner Transcripts]**。
5. 選擇 **[!UICONTROL Generate New]**。
6. 選擇您需要產生逐字稿的日期範圍。 你可以透過 **[!UICONTROL Choose dates]** 日期範圍下拉選單中的選項修改開始和結束日期。
7. 請選擇以下內容：
   1. 從該 **[!UICONTROL Select Learners]** 區段選擇學習者的名字。 你可以選擇使用者或使用者群組，或複製貼上你想產生成績單的學習者的電子郵件地址。 詳情請參閱「使用複製貼上產生學習者成績單[&#128279;](#generate-learner-transcript-using-copy-paste)」章節。若未選擇，則預設為「全部值」。
   1. 從 **[!UICONTROL Select Catalogs]** 下拉選單中選擇特定目錄。 文字稿僅為指定目錄下載。 若未選擇，則預設為「全部值」。
   1. 選擇 **[!UICONTROL Enrollment Status]**。 此下拉選單包含以下選項：

      * 全部選擇
      * 完成
      * 進行中
      * 未開始
      * 未註冊

8. 進階選項：選擇 **[!UICONTROL Advanced options]** 下載逐字稿以包含以下內容：

   1. 請選擇勾選 **[!UICONTROL Include deleted Learners]**&#x200B;方塊，下載已被刪除帳號的學習者成績單。
   1. 透過啟用 **[!UICONTROL Enable module level information]** 勾選框，下載學員成績單中的模組層級資訊。 若啟用此功能，模組名稱及每個模組所花費的時間會作為逐字稿的一部分被擷取。
   1. 透過啟用選項 **[!UICONTROL Include skills data and summary sheets]** 勾選框，下載技能資料與摘要表。 更多資訊請參閱 Excel 報告區塊。

9. 您也可以選擇要填入報告的欄位。 此功能提供彈性，可依需求下載特定欄位值的報告。 從下拉選單選擇欄位。
當技能資料未包含時，成績單會產生並下載為.zip檔案。 若啟用技能資料勾選框，成績單會產生並下載為 。 XLX 檔案。

### 使用複製貼上方式產生學習者文字稿

取得學習者成績單變得繁瑣，因為只能逐一取得學習者或使用者群組。 在這裡，利用複製貼上功能，你可以一次性複製學習者電子郵件 ID 清單並貼上。

1. 選擇 **[!UICONTROL Email IDs]** 分頁輸入複製的獨一無二電子郵件 ID 清單。
2. 貼上你想新增學習者的專屬電子郵件 ID，中間用逗號、分號或換行。
3. 選擇 **[!UICONTROL Validate Emails Ids]** 檢查您輸入的電子郵件 ID 是否有效。 若輸入的電子郵件 ID 錯誤，該郵件會以紅色標示，並附上驗證訊息。

   >[!NOTE]
   >
   >除非輸入的所有電子郵件 ID 都正確，否則「生成」按鈕將保持停用狀態。

4. 選擇 **[!UICONTROL Generate]** 為所有提及的電子郵件 ID 產生學習者成績單。

   >[!NOTE]
   >
   >產生學習者成績單可結合使用者與電子郵件識別標籤下的電子郵件識別碼。

### 學習者成績單報告包含哪些內容

學習者成績單報告結合使用者、註冊及學習物件相關資訊。
以下表格描述每種類型：

**使用者相關資訊**

以下欄位用以識別學習者。

| 場地 | 說明 |
|---|---|
| 名稱 | 學習者的姓名。 |
| 電子郵件 | 學習者的電子郵件地址。 |
| Adobe ID | 此欄位僅在使用者使用 Adobe ID 登入時才會被填入。 若透過組織定義 [的單一登入（SSO）](/help/migrated/administrators/feature-summary/multiple-sso-logins.md)存取 Adobe Learning Manager，Adobe ID 欄位將保持空白。 |
| 使用者唯一識別碼 | 使用者唯一識別碼是由帳號產生的外部識別碼，以防帳號沒有所有使用者的電子郵件 ID 或所有使用者的唯一電子郵件 ID。  <br>使用者唯一識別碼欄位是可選的欄位，可以啟用給帳號使用。 該欄位的主要目的是讓帳號能以獨特 ID 標記每位使用者以追蹤他們、透過 API 更新使用者紀錄、審計或在自動化工作流程中同步資料。 每個使用者的標籤都是透過 CSV 匯入來完成的。</br><br>若帳號選擇了「獨特使用者ID」，則報告（如學習者成績單）會在報告中提供欄位。</br> |

**註冊相關資訊**

以下欄位記錄活動、進度或嘗試。

| 場地 | 說明 |
|---|---|
| 註冊日期（UTC時區） | 學習者註冊的日期與時間戳記，為學習物件類型，該類型為課程、認證或學習路徑。 |
| 標記 完成日期（UTC時區） | 教師標記課程或模組完成的日期與時間戳記。 請注意，如果會議尚未發生，該欄位在報告中會顯示為空白。 此外，若課程已進行且講師未標記為完成，報告中該欄位會顯示空白。 |
| 開始日期（UTC時區） | 學習者開始學習物件的日期與時間。 空表示學習者尚未開始此步驟。 |
| 完工日期（UTC時區） | 學習者完成此項的日期與時間。 空表示學習者尚未完成此項。 |
| 標記 完成日期（UTC時區） | 記錄講師標記課程或模組完成的精確日期與時間。 |
| 截止時間（UTC時區） | 學習者應在完成此學習對象的日期與時間。 空的意思是沒有截止日期。 |
| 逾期 | 已註冊於學習對象的學習者目前逾期狀態。 是/不是 |
| 現況 | 顯示學習者在修讀課程、認證或學習路徑時的狀態。  可用狀態為未開始、未註冊、進行中或已完成。 |
| 進步百分比 | 學習者目前修習課程、認證或學習路徑的進度百分比。 |
| 花費時間（分鐘） | 學習者在 LO 中所花費的學習時間，模組層級的列顯示各模組的學習時間所花費。 課程/學習路徑/證書等級的列顯示累積的學習時間。 |
| 等級 | 表示學習者的成功。 「通過」，如果使用者符合成功標準，則「失敗」。 |
| Quiz_score | 學習者獲得的最新測驗分數。 如果學習者沒嘗試過測驗，或內容沒有測驗，或管理員/講師沒有給分，可能會是空的。 該欄用來記錄最近一次測驗嘗試的分數。 例如，如果使用者嘗試多次（例如三次嘗試中得10、50和30分），Quiz_score欄會顯示上次嘗試的分數，即30分。 假設一個測驗的最高分數是100分，使用者嘗試三次，分別得分30、60和90分。 Quiz_score欄顯示90分（最近一次分數），Highest_Quiz_score欄顯示90分（所有嘗試中最高分），Quiz_score_max仍為100分（最高分數）。 |
| Quiz_score_max | 本模組最新的最高測驗分數。 如果學習者沒嘗試過小考，或內容中沒有任何小考，可能會是空白的。 Quiz_score_max欄代表特定測驗或模組中可達到的最高分數。 由於Quiz_score_max保持不變，報告中顯示測驗或模組的總分數非常有用，無論使用者表現如何。 |
| Highest_Quiz_score | Highest_Quiz_score欄代表使用者在所有特定測驗嘗試中所取得的最高分數。 例如，若使用者嘗試三次，分別得分為10、20和15，Highest_Quiz_score會顯示20分，因為這是最高分數。 |
| Highest_Quiz_score_max | 該模組最高測驗分數。 如果學習者沒嘗試過小考，或內容中沒有任何小考，可能會是空白的。 學習者在多次嘗試中所取得的最高測驗分數所可能的最高分數。 這並不是學習者取得的最高分數。 它會捕捉學習者在嘗試中最高分數時可能達到的最高分數。 |
| 嘗試 | 學習者迄今為止完成本模組的總嘗試次數。 |
| 最大允許嘗試次數 | 最大嘗試次數允許學習者使用模組。 |
| 投稿留言 | 學習者經理完成學習物件後的評論。<br>講師提供的提交意見資料包含在檔案提交模組中。 更多資訊請參閱 <a href="https://experienceleague.adobe.com/zh-hant/docs/learning-manager/using/instructor/modules#filesubmissionforactivitymodules">模組-Adobe Learning Manager。</a></br> |
| 完結來源 | 指的是學習者完成課程、學習路徑或認證的起源或方法。 它幫助管理員了解完成過程或系統記錄的方式。 欄位標示完成是自我申報，或是由特定角色或配置促成。 注意：對於 VC 連接器的出席工作流程，當學習者自動被標記為已出席時，來源會顯示「SELF， &lt;learner_email>」。&lt;/learner_email> |
| 完成評論 | 管理員在完成課程、認證或學習路徑後，將學習者標記為完成時所發表的評論。 管理員可以為一位或多位學習者新增完成評論。 |

**學習物件相關資訊**

這些指的是課程、模組、學習路徑、認證等。

| 場地 | 說明 |
|---|---|
| 學習計畫名稱 | 學習計畫標題。 |
| LP/認證/課程 | 學習對象的標題。 |
| 類型 | 使用者所註冊的學習物件類型。 例如，<ul><li>學習路徑</li><li>認證</li><li>河道</li></ul> |
| 嵌入路徑 | 嵌入式路徑是一種包含於其他課程或學習路徑中的學習路徑。 該欄位表示學習者是作為另一條學習路徑的一部分完成該學習路徑，而非作為獨立作業。 |
| 河道 | 當然是使用者註冊的名稱。 當該列為空時，代表認證或學習路徑。 <br><b>注意：</b> 雖然學習路徑與由獨立課程或巢狀學習路徑組成，但每個組件仍保留獨立紀錄。 這確保父元件與子元件的進度、完成度及報告資料都能分別追蹤。</br> |
| LO 唯一識別碼 | 這是 Adobe Learning Manager 中學習物件（課程、認證或學習路徑）的可選管理員指派識別碼。 它主要用於維護自身外部系統 ID 以管理學習內容的組織，並希望將這些 ID 映射到 ALM 學習物件以進行整合或報告。 LO 唯一 ID 只有在帳號啟用此功能且作者在建立 LO 時指派了 ID 時才會顯示。 注意：訓練識別碼始終存在，並唯一識別 ALM 中的每個學習對象。 LO 唯一識別碼用於跨系統映射，標準 ALM 操作並非必需。 |
| 執行個體 | 註冊的實例名稱是學習物件使用者的。 |
| 納入標準 | 此欄位顯示學習者如何註冊學習對象（課程、認證或學習路徑）。 其價值的確定方式如下：<ul><li>管理員/經理註冊：當學習者由管理員或經理直接註冊時，會直接顯示。 </li><li>學習計畫註冊：當學習者透過學習計畫或自動註冊觸發器註冊時，顯示自動註冊。</li><li>管理員註冊使用者群組：若學習者已註冊為使用者群組成員，則顯示使用者群組名稱。 </li><li>巢狀學習路徑：若學習路徑 1 包含學習路徑 2，而學習路徑 2 包含課程 A：對於 LP2 與課程 A，則值為父路徑。 對於LP1，其值為直接值。 </li><li>自我註冊：當學習者自行註冊時展現自我。 </li></ul>本欄的數值反映實際的註冊方式與 LO 階層，如上所述。<ul><li>學習計畫註冊學習者：價值：自動註冊 學習者會透過學習計畫或自動註冊觸發器自動註冊。 </li><li>學習者自我報名：價值：自我 學習者可直接報名課程、認證或學習路徑。 </li>行政人員直接註冊學員（使用學員電子郵件/姓名）：價值：直接。 管理員或經理會手動指定學習者的電子郵件或姓名來註冊。 <li>透過使用者群組註冊：值：使用者群組名稱 學習者是作為使用者群組的一員註冊的。 若學習者屬於多個使用者群組，報告將顯示該註冊所經過的相關使用者群組。  </li><li>因註冊學習路徑而註冊的學習物件：價值：路徑。 學習者之所以選修課程或模組，是因為它是他們被分配到的更大學習路徑的一部分。</li></ul> |
| 模組 | 課程內的模組名稱。 報告中僅顯示已完成或進行中狀態的模組。 如果狀態是未啟動或未註冊，模組欄位會保持空白。<br>請選擇「啟用模組層級資訊</b>」勾選框，在學習者成績單<b>中下載模組層級資訊。若啟用此選項，模組名稱及每個模組所花費的時間會作為逐字稿的一部分被擷取。</br> |

### 學習摘要 I 表包含哪些內容

追蹤正在使用的學習路徑、課程或認證。 追蹤進行中的活動以及即將到來的訓練截止日期。

* 註冊學習者人數：表示特定學習對象（課程、學習路徑或認證）已註冊多少學習者，無論是否已開始。
* 已開始學習人數：顯示已啟動或開始課程、認證或學習路徑的學習人數。
* 完成培訓的學習者人數：顯示成功完成訓練並符合所有完成標準的學習者人數。
* 進步學習者人數 ≥ N%：反映在培訓中達成至少指定進度門檻（例如70%）的學習者數量，即使他們尚未完成訓練。
* N天內截止日期的學習者人數：表示未來「N」天（例如7天）內有多少學習者需接受培訓，有助於識別即將到來的截止日期。

### 如何解讀資料

我報告的學習摘要追蹤了分配給學習者的兩條學習路徑。

* 使用者已註冊兩條學習路徑，並已開始兩條路徑。
* 這兩條學習路徑都還沒完成。
* 學習者在任一路徑中尚未突破70%門檻。
* 接下來七天內，兩項訓練都沒有截止日期。

### 學習摘要 II 表格包含什麼內容

追蹤每位學習者的學習活動。 追蹤學生的註冊情況、進行中的活動及截止日期。

* 註冊學習物件數量：學習者在每門課程、認證或學習路徑中註冊的學習物件（LO）總數。
* 啟動學習物件數量：表示學習者已啟動或啟動多少已註冊的學習物件。
* 完成的學習對象數量：顯示學習者已完成多少個已啟動的學習對象。
* 進步≥學習對象數量 N%：反映學習者達成至少指定進度門檻（此例為70%）的學習對象數量。
* N天截止日期的學習物件數量：識別在接下來設定天數內（此例為7天）內截止的LO，有助於追蹤截止日的接近。

### 如何解讀資料

* 學習者註冊兩個學習對象，且兩者皆已開始。
* 尚未完成任何學習對象。
* 學習者在任何一項中都尚未達到70%的進步率。
* 這些學習項目都沒有在接下來的7天內截止。

## 學習者成績單下載歷史

學習者成績單下載歷史讓管理員能追蹤誰下載了每位學習者的成績單，以及他們存取了哪些使用者。 此功能在企業及合規導向的環境中特別有用，因為多個利害關係人可能需要查看學習紀錄。

下載學習者成績單後，學習者成績單頁面會列出平台上任何人產生的所有成績單。

該清單顯示以下屬性：

* 發件人與收件人：待下載的逐字稿長度。
* 產生者：下載報告或申請下載的使用者電子郵件。
* 學習者：需下載其成績單的學習者或學習者團體。
* 套用篩選條件：已套用於註冊狀態的篩選條件。
* 新增資料：管理員在「新增學習者成績單」模式的進階選項中請求的額外資料（刪除學習者、模組資訊、技能資料與摘要表）
* 狀態：已下載、排隊或進行中。
* 取消：隨時取消報告產生。

## 學習者成績單的額外考量

學習者成績單包含幾項管理者應注意的行為：

**學習路徑與課程進度顯示**

所有屬於學習路徑（LP）的課程都會顯示在學習者成績單中，即使學習者僅在其中一門課程中取得進展。 這確保即使進度僅部分完成，也能完全看到學習路徑。

**刪除學習者的資料**

若學習者已被平台刪除，其成績單將不會顯示在所屬用戶群組的任何報告中。 這表示刪除學習者的紀錄將被排除在任何使用使用者群組篩選器建立的報告之外。

但你仍然可以下載被刪除學習者的資料。 如果你在設定篩選器產生報告時選擇了這個選項 **[!UICONTROL Include deleted learners]** ，就可以下載已刪除學習者的報告。

**自訂管理員的行為**

具有明確範圍的自訂管理員（例如，僅限於特定目錄或使用者群組）會根據其範圍看到經過篩選的逐字稿資料：

* 使用者群組範圍：報告中僅包含自訂管理員範圍使用者群組中的學習者。
* 目錄範圍：僅指派至範圍目錄的學習對象（課程、認證及學習路徑）將納入報告中。
* 篩選欄位：欄位保持不變。 只有列會根據目錄範圍和使用者群組來設定範圍。

這確保有範圍的自訂管理員只查看他們被授權管理的學習者資料與學習內容。

**連接器支援**

學習者成績單報告可透過管理員使用者介面、 [FTP、Box、工作 API 或 Power BI](/help/migrated/integration-admin/feature-summary/connectors.md) 存取。 它未包含在 Salesforce、Power BI 和 Marketo Engage 的統一報告中。

從 Salesforce、Marketo Engage 和 Power BI 下載的統一報告欄位數比學習者逐字稿少。
