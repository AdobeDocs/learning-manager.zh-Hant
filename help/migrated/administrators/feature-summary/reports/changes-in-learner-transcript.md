# 四月發布中學習者成績單的變更

## 補全方法欄位

**完成方法**&#x200B;欄位顯示管理員學習者成績單中每筆紀錄的填寫方式。

**價值觀：**

1. **直接完** 備（用於直接完備）
2. **替代** （指透過替代關係達成的完成）
3. **替代撤銷** （當所有替代完成因追溯未完成及關係移除而被撤銷）

>[!NOTE]
>
>此欄在學習者的LT中無法顯示;它僅在行政LT中提供，用於報告和追蹤。

**影響**：讓管理者能有清晰的稽核追蹤、合規追蹤，以及課程完成過程的透明度。

## 學習者成績單中的替代完成追蹤

替代完成允許學習者在完成等效來源課程或路徑後，根據已建立的關係，獲得目標課程或學習路徑的完成學分。

在學習者成績單（LT）中，替代完成項會影響三個現有欄位： **狀態**、 **完成日期**&#x200B;及 **完成來源**。

- **狀態**：即使學習者因替代完成而未直接完成目標課程或路徑，該狀態仍可為 **完成** 。 其他狀態（**未開始**、 **進行中**、 **未登記**）則不受影響。
- **完成日期**：替代完成日期則繼承自原始課程或路徑。 若學習者後來直接完成目標，日期會更新以反映直接完成。
- **完成來源**：擷取提供替代完成的來源課程或路徑的訓練ID。 多個有效來源以逗號分隔的 ID 列出。 若來源被撤銷，則只剩下活躍的來源。 當存在多個來源時，則使用最早的完成日期。

**影響**：替代完成減少人工對帳，自動化學習路徑與認證進度追蹤，並支援合規要求。

>[!NOTE]
>
>學習者成績單不會顯示 **完成方法** 欄位。 本欄僅在行政學員成績單中提供。

## 備選的完工日期邏輯

**學習者成績單中的完成日期**&#x200B;欄位用於直接完成及替代完成。

- 對於替代完成課程，日期反映學習者完成原始課程或路徑的時間。
- 若學習者後來直接完成目標，完成日期會更新為直接完成日期。
- 未新增替代完工日期欄位。
- 若多個來源提供替代完成日期，則使用最早的有效完成日期。
- 若來源被撤銷（啟用追溯未完成），日期更新至下一個最早的活躍來源，若無活躍來源則被清除。

**影響**：確保準確的歷史追蹤與一致的報告，即使替代關係隨時間改變。

## 撤銷的替代完成計畫

撤銷的替代完成是指目標課程或學習路徑的所有來源關係被移除，前提是啟用追溯性未完成功能。

### 觸發條件

- 追溯未完成必須在帳戶層級啟用。
- 撤銷僅在&#x200B;**&#x200B;**&#x200B;所有有效來源關係被移除時發生。
- 若至少有一個來源仍存在，則替代補全會持續存在，補全來源欄位也會相應更新。

### 對學習者成績單的影響

- **狀態**：若所有替代完成都被撤銷且無直接完成，狀態會更新（例如從 **完成** 變 **為未開始** 或 **進行中**）。
- **完成日期**：若無活躍來源且無直接完成，則已清除。
- **完成來源**：已更新以移除已撤銷的來源;若所有來源均被撤銷，則清除。

若學習者有直接完成，撤銷替代來源不會影響完成狀態或完成日期。

>[!NOTE]
>
>若多個來源提供替代完成，且僅部分被撤銷，逐字稿會反映剩餘的活躍來源及其最早完成日期。 若所有來源皆被撤銷且無直接完成，學習者將失去目標的完成狀態。

## 檢視清單審查員意見的強化報告

此變更統一適用於所有 Admin LT 來源（UI 匯出、工作 API 報告及適用的連接器）。 連接器匯出的 LT 會將審查者的評論作為專屬欄位出現在末尾（針對先前未公開提交意見的連接器），確保後續整合能區分審核者意見與其他評論。

>[!NOTE]
>
>對於學習者逐字稿，先前標示 **為提交評論** 的欄位現在改名為 **評審者備註**，並在啟用時顯示評審員的留言清單。

## 改進學習時間計算

學習者逐字稿報告現在採用更精細的邏輯，根據使用者活動和瀏覽器分頁的重點區分活躍與閒置學習時間。

**影響**：提供更精確的學習者參與度衡量，支援合規報告與分析。
