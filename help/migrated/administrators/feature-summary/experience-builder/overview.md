# 概觀

Experience Builder 是 Adobe Learning Manager 中的一個無程式碼/低程式碼工具，幫助你建立客製化的學習入口網站。 它讓你能設計出品牌化、使用者友善的學習入口網站，無需具備技術能力或豐富的程式設計知識。

透過體驗建構器，管理員可以輕鬆建立頁面、選單和小工具，提供符合受眾需求的個人化學習體驗

許多組織在沒有技術協助或昂貴的系統整合商的情況下，難以自訂他們的學習入口網站。 他們希望入口網站能與品牌相符、提供精準內容，並能適應不同學習族群，同時又能快速且容易建置。

## 經驗建構器解決的問題

Experience Builder 針對組織在無需重大技術協助或昂貴系統整合商的情況下，客製化學習入口網站時面臨的常見挑戰。 它彌合了兩個主要選項之間的差距：

* 標準的開箱即用體驗，提供有限的自訂選項，且可能讓每個學習入口看起來都相似。
* 無頭實現允許完全客製化、遊戲化的入口網站，但也帶來重大挑戰，包括上市時間長（通常3-6個月）、依賴開發團隊及高昂成本。

Experience Builder 提供中間路線，允許建立品牌入口網站與獨特的學習旅程，且避免無頭式方法的高昂成本與開發負擔。

## 使用經驗建構器的主要好處

使用頁面、小工具或選單的功能性帶來幾項關鍵好處：

* **輕鬆自訂**：你可以設計精準符合品牌的入口網站，搭配自訂的頁首、頁尾、標誌和版面配置。 注入自訂 HTML、CSS 與 JavaScript 的能力，提供完整的品牌體驗。
* **個人化學習**：一大優勢是能為不同學習者提供不同的體驗。 你可以為特定使用者群組（例如銷售團隊、設計師或工程師）設定頁面和選單，確保他們只看到與自己相關的內容。 這非常適合為特定團隊或時期打造針對性的微學習活動。
* **無程式碼/低程式碼介面**：管理員可以透過導引式、視覺化的介面創造學習體驗，而非直接撰寫程式碼。 這種方式讓管理員能在不需技術能力或豐富程式設計的情況下，建立和管理先進且具品牌特色的入口網站。
* **全球化與行動裝置相容**：Experience Builder 支援建立多語言頁面，滿足多元全球受眾的需求，並設計為行動裝置友善，確保桌面、手機及平板電腦的學習者能享有順暢體驗。

## 常見使用案例

Experience Builder 可用於各種涉及內部員工與外部使用者的學習情境。

* **基於職務的培訓入口網站**：具有特定部門培訓需求的組織，例如擁有獨立銷售與客戶成功團隊的金融公司，可為每個團隊建立專屬學習頁面，確保內容高度相關。
* **活動專屬學習頁面**：你可以為企業活動（如科技高峰會或銷售啟動會）建立臨時且專門的頁面。 這些頁面可包含會議資訊、講者名單及活動行事曆小工具，且可僅針對相關團隊在特定期間內提供，之後恢復標準入口網站體驗。
* **客戶學院**：Experience Builder 讓代理商打造反映品牌識別的客戶面向學院，實現客製化體驗，且不需花費無頭打造所需的時間與成本。

## 認證的外部入口網站工作流程

使用 Experience Builder 建立的面向客戶的學院，完全由 Adobe Learning Manager 管理。 這些入口網站採用 Adobe Learning Manager 內建的認證、權限與安全框架。

每位外部學習者必須登入 Adobe Learning Manager，並至少加入一個使用者群組。 目前，Experience Builder 不支援未經認證或面向公眾的入口網站。 所有個人化體驗都需要學習者登入 Adobe Learning Manager。

管理員可以使用體驗建構 **[!UICONTROL Menu]** 器選項，將自訂建立的頁面指派為特定使用者群組的登陸頁。 當該群組的學習者登入時，Adobe Learning Manager 會自動引導他們到指定的登陸頁，為該受眾打造個人化且具品牌特色的體驗，例如客戶培訓、合作夥伴賦能或新手導入。

### 需求與限制

* 需驗證：Adobe Learning Manager 中僅對已認證的使用者開放個人化內容、自訂頁面與選單。
* 使用者群組分配：學習者必須加入正確的使用者群組，才能存取指定的登陸頁面與選單。
* 群組式登陸頁：著陸頁設定適用於使用者群組的所有成員，確保相似受眾的體驗一致。
* 自訂範圍：Experience Builder 支援使用 widgets、HTML 與 iFrame 進行廣泛的 UI 與版面自訂。 然而，進階整合如電子商務、聯邦單點登入或外部資料連線可能需要混合或無頭實作。

### 面向外部的入口網站設定工作流程

* 定義使用者群組：在 ALM 中建立或識別代表外部受眾的群組（例如客戶、合作夥伴或經銷商）。 [欲了解更多使用者群組資訊，請參閱 Adobe Learning Manager](/help/migrated/administrators/feature-summary/user-group.md) 中的使用者群組。
* 將學習者分配到群組：將每位外部學習者加入適當的使用者群組，讓他們登入後能被導向正確的入口網站體驗。
* 設計入口頁面：使用體驗建構器建立帶有 Adobe Learning Manager 元件、HTML 和 iFrame 元件的品牌頁面。 欲了解更多資訊，請參閱 [Experience Builder](/help/migrated/administrators/feature-summary/experience-builder/create-a-page.md) 中建立自訂頁面。
* 設定選單與登陸頁：在選單建立器中，為每個使用者群組指派一個獨特選單，並將其自訂入口頁面指定為登陸頁。 查看 [建立選單](/help/migrated/administrators/feature-summary/experience-builder/create-a-menu.md) 以獲取更多資訊。
* 測試與發布：在發布入口網站前，請確認每個使用者群組的導航、內容可見性及頁面路由。
