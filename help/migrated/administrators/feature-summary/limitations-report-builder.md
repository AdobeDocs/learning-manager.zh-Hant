# Adobe Learning Manager 中報告建構器的限制

## 概觀

報表建工具涵蓋廣泛的自訂報表需求，但本版本不支援部分報表類型、資料集及組態。 本節列出已知限制，並在可用時建議解決方法。

## 反覆認證

本版本中，某些涉及重複認證的查詢在報告建置器中無法正常運作。 如果您的報告涉及重複認證週期，例如追蹤多個認證期間的重新註冊或重修，請改用 Adobe Learning Manager 中的現有認證報告。

## 不支援的資料集與報表類型

以下資料在本版本的報告建器中未提供：

* **商業成果資料**：報告建構器不支援自帶資料（BYOD）整合。 不支援結合外部商業資料與 LMS 資料的報告。
* **稽核報告**：系統稽核軌跡和管理員活動日誌無法在報告建構器中建立。 請在管理員檢視中使用現有的稽核報告來取得這些資料。
* **增量報告**：報告建構器每次都會產生完整的快照報告。 增量報告只回傳自上次下載以來變更的紀錄不支援。 對於增量資料，請使用 Job API。

## 趨勢報告的限制

### 每份趨勢報告僅有一個日期欄位

單一趨勢報告只能依一個日期欄位分組。 你不能在同一份報告中同時呈現兩個不同的日期趨勢。

例如，你可以建造：

* 按月份&#x200B;**顯示**&#x200B;報表（依登記日期分組）
* 另一份按月份&#x200B;**顯示**&#x200B;完成度的報告（依完成日期分組）

你無法建立一份報告，同時顯示註冊月份和完成月份作為獨立的趨勢欄位。 要同時取得兩者，請建立兩份報告並並排分析。

### 趨勢反映的是當前數據，而非歷史快照

報告建構器中的趨勢報告是基於當前的點點資料，而非歷史快照。 報告反映目前紀錄的狀況，分布於各日期欄位。

這意味著：

* 一月註冊但後來退選的學習者，可能不會出現在一月的註冊趨勢中
* 重新組織的用戶群組不會反映過去幾個月的歷史會員數
* 被刪除的紀錄可能不會出現在其活躍期間的趨勢中

使用趨勢報告進行方向分析。 為了精確的歷史紀錄或稽核目的，請使用現有的固定報告或工作 API 匯出。
