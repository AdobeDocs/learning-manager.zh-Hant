# 報表建構器中的可用資料集

## 概觀

報表建工具將所有可用的欄位組織成資料集，並以相關欄位命名的群組。 本文列出每個資料集，說明其內容，並註明哪些資料集可合併成單一報告。

## **可用資料集**

表格中的資訊並非完整。 完整列出資料集中所有欄位，請參閱「資料集中&#x200B;**欄位列表」一節**。

| **資料集** | **內容** | **關鍵欄位** |
|----------------------------------------------------------|--------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **使用者** | 活躍與已刪除學習者的學習者檔案資料 | 姓名、電子郵件、使用者ID、經理、活動欄位、狀態 |
| **成績單（學習物件）與成績單（模組）** | 入學與完成紀錄 | 註冊日期、完成日期、進度百分比、狀態 |
| **學習物件** | 課程、學習路徑與認證元資料 | 學習物件名稱、學習物件 ID、類型、目錄、目錄標籤、狀態 |
| **學習物件實例** | 多實例課程的實例層級細節 | 實例名稱、實例 ID、註冊上限、截止日期 |
| **目錄** | 目錄元資料與目錄標籤鍵值對 | 作者姓名、創作日期、時長、格式。 **目錄標籤欄位由客戶自行設定**。 只有當你的帳號有設定目錄標籤時，才會出現。 你看到的標籤名稱和數值會反映你自己的設定。 |
| **使用者群組** | 使用者群組成員與階層 | 創建日期、會員數量、姓名、狀態 |
| **模組** | 課堂、虛擬教室及電子學習模組的課程詳情 | 模組編號、講師姓名、開始時間、結束時間、地點 |


>[!NOTE]
>
>對於透過目錄共享從其他帳號取得的課程， **學習物件資料集中的作者名稱** 欄位會回傳原始作者名稱。 這與標準的 ALM 管理介面及固定報告不同，後者所取得課程會顯示「外部作者」。 此行為僅適用於報告建構器，且僅適用於對等（接收）帳戶。

## 資料集如何連結

並非所有資料集對都能合併成單一報告。 資料集必須在 Adobe Learning Manager 的資料模型中共享邏輯關係，才能被加入。 當你新增第一欄時，報表建構器會篩選剩餘的資料集，只顯示相容的。 如果資料集顯示為灰色，就無法直接與你已選取的欄位連接。

這表示該資料集無法與你已選擇的欄位合併。 這是資料模型中的一個硬性限制。 這兩個資料集沒有共用相容的連接路徑。

一個常見的例子是 **「合規率** 」衍生欄位。 當選擇合規百分比時， **逐字稿**、 **模組逐字稿**&#x200B;及 **LO 實例** 資料集皆被停用。 合規率會在使用者或使用者群組層級針對學習物件與目錄計算。 它僅設計與使用者&#x200B;**、使用者群組**、**學習物件****及目錄**&#x200B;欄位及篩選器一起使用&#x200B;****。

要使用被停用的資料集，先取消選取造成衝突的欄位，然後加入你需要的資料集。

>[!NOTE]
>
>如果產生的報告包含課程資料但沒有學習路徑資料，請檢查你是否將學習物件資料集與模組資料集合併。 模組直接連結到課程，但不直接連結到學習路徑。 因此，當包含模組欄位時，報告建構器可能只回傳課程紀錄，並過濾學習路徑。 若要報告學習路徑，除非報告專門用於分析課程層級的模組資料，否則避免新增模組欄位。

以下的連接關係取自報表建置器資料模型。 了解它們有助於你規劃要合併哪些資料集，再建立報告。

### 集線器資料集

兩組資料集作為中央樞紐。 大多數其他資料集則透過這些資料集相互連結：

* **註冊** （註冊資料集），主要事實表。 它直接連接到 **使用者** （註冊的學習者）、 **學習物件** （他們註冊的對象），並透過學習物件連接到 **模組**、 **目錄**、 **目錄標籤**&#x200B;和 **LO 實例**。
* **模組逐字稿** （moduleTranscript 資料集）— 模組層級進度事實表。 它連接 **User** 和 **Module，而 Module** 又連結到 **Learning Object**。

大多數結合學習者、課程與完成度資料的報告，都是透過這兩個中心之一建立的。

### 別名關係

資料模型中的某些欄位透過命名角色與 **使用者** 實體連結，而非直接的學習者 ID。 這些是別名外鍵。 它們指向相同的使用者表，但代表不同的角色：

* **講師**：模組加入使用者，成為該課程的講師
* **作者**：學習物件作者作為學習物件的作者加入使用者
* **管理器**：使用者自我連結，代表學習者的管理器
* **完成者**：招生與模組文字稿各附一個「完成者」使用者參考文獻

這就是為什麼一份報告可以同時顯示學習者的姓名和講師的名字。 它們都是透過不同的連接路徑從使用者實體傳送來的。

### 使用者群組資料集類型

**使用者群組**&#x200B;類別包含數個不同的資料集視圖，每個檢視涵蓋不同類型的群組：

| **資料集** | **群組類型** |
|---------------------------------------------------------|---------------------------------------------------------------|
| **使用者群組** （userGroup） | 所有使用者群組。 將此作為主要使用者群組資料集 |
| **使用者群組（活躍領域）（** active_field_user_group） | 根據活躍領域值（例如地區、部門）劃分的群組 |
| **使用者群組（團隊）（** team_user_group） | 基於管理者階層的群組 |
| **使用者群組（自訂）（** custom_user_group） | 手動設定自訂群組 |
| **使用者群組（內建）（** inbuilt_user_group） | 系統定義群組 |

基於視圖的使用者群組資料集（**Active Field**、Team **、****Custom**、**Inbuilt**）在結構中沒有直接的外鍵關聯，它們是 SQL 視圖。這表示它們的連接限制比核心 **使用者群組** 資料集寬鬆。 將用戶群組資料與登記或逐字稿資料合併時，請使用核心 **使用者群組** 資料集，以獲得最可靠的結果。

## 衍生欄位

衍生欄位是由報表建構器預先計算的欄位。 它們在欄位選擇器中會與標準欄位分開列出。

| **導出域** | **它計算的內容** | **所需資料集** |
|-------------------------|-------------------------------------------------------------------------|-----------------------------------------|
| **遵從率** | 完成合規標籤課程的必修學習者比例 | 使用者群組、學習物件、逐字稿 |
| **完成率** | 完成次數除以註冊人數 × 100 | 逐字稿或學習物件 |
| **註冊用戶數** | 學習對象的總註冊學習者數 | 學習物件 |
| **完成次數** | 學習對象的總完成次數 | 學習物件 |
| **起跑計數** | 已開始但尚未完成的學習者 | 學習物件 |
| **成員人數** | 使用者群組中的使用者數量 | 使用者群組 |
