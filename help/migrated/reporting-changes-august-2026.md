# 2026 年 8 月 Adobe Learning Manager 版本中的報告變更

2026 年 8 月的 Adobe Learning Manager 版本將新增涵蓋自適應課程、成績冊、外部學習、Gen AI 學分使用等多項報告功能。 本文總結了本版本中管理員可新增的欄位、報告及行為變更。

## 改變了什麼

報告更新涵蓋八大功能領域：自適應課程行為、自適應候補名單、成績簿評分、外部學習、增量用戶輸出、Gen AI 學分使用、根認證追蹤，以及 webhook 時間戳對齊。 這些變更對以下報告影響最為重大：

- 學習者成績單（LT）
- 訓練報告
- 招生報告
- 候補名單報告
- 內容審核報告

大多數更新都會新增欄位。 有些則引入了新的報告類型。 其中幾項改變了現有資料的建模或格式化方式。

## 自適應航向報告變更

### 訓練報告

訓練報告新增三欄支持適應性課程行為。

| **柱狀** | **描述** | **支持價值** |
|--------------------------|----------------------------------------------------------|------------------------------------------------------------------------|
| 自適應學習物件 | 判斷賽道是否具適應性 | 真（自適應）、假（非自適應） |
| 可見性使用者群組 | 列出可查看每個模組的使用者群組 | 一個或多個使用者群組名稱（例如，All Learners，UG-Australia） |
| 強制性 | 表示某模組是否對使用者群組為必修 | 模組必須的使用者群組名稱;空白 = 可選 |

你可以將可視性使用者群組&#x200B;**與**&#x200B;強制&#x200B;**性組合結合**，直接在報告中解讀自適應完成規則。例如，某模組可能對 **所有學習者** 開放，但僅對 **管理員組**&#x200B;必須。

### 學習者成績單

當自適應邏輯觸發補全時，新增一個「 **先前完成」** 欄位捕捉歷史完成資料。

| **子領域** | **描述** |
|-----------------------|-----------------------------------------|
| 完成更新日期 | 完成重置的時間戳記 |
| 完成日期 | 先前完成時間戳記 |
| 更新進度 | 學習者重置前的進度 |
| gradeAtRefresh | 重置時的學習者分數 |

學習者成績單現支援多次完成循環。 例如，當因課程更新或新增必修模組而發生重修事件時，先前完成的課程會移至 **先前完成欄位** 。 目前的完成項目仍位於標準成績單欄位中。

### 招生報告

新增 **的候補名單** 欄位會顯示學習者是否在課程中的任何模組被列入候補名單。

| **價值** | **意義** |
|-----------|---------------------------------------------------------|
| 沒錯 | 學習者在一個或多個模組中被列入候補名單 |
| 錯誤 | 學習者已確認已註冊所有可見模組 |

### 候補名單報告

新增兩個欄位及強化的狀態細節支援模組，能在模組層級追蹤候補名單。

| **柱狀** | **描述** |
|-----------------|----------------------------------------------------------------------------------------------------------------------------------------|
| **模組** | 學習者被列入候補名單的模組名稱（教室或虛擬教室課程）。 出現在實例狀態欄之後。 |
| **模組識別碼** | 學習者被列入候補名單的模組識別碼。 出現在模組欄位之後。 |
| **嵌入** | 包含此課程的任何學習路徑名稱與 ID。 如果課程不是學習路徑的一部分，則會被填空。 |

候補名單報告已從課程層級模型轉變為模組課程層級模型。 學習者現在可以在部分模組註冊，其他模組則可被列入候補名單。 報告也支援彈性學習路徑內的候補名單追蹤，該路徑在模組層級執行名額限制。

### LP註冊報告

學習路徑註冊報告也新增了 **備註** 欄。 當學習者在任何課程或虛擬教室課程中處於候補狀態時，備註欄會顯示 **「候補**&#x200B;名單」。 當所有會議都確認時，該欄為空白。

### 出席人數報告

**學員狀態**&#x200B;欄位現在區分已確認與候補學員。

| **價值** | **意義** |
|------------|----------------------------------------|
| 已確認 | 學習者擁有分配的名額 |
| 候補名單 | 學習者等待名額分配 |

## 成績簿報告變更

### 學習者成績單

新的 **權重** 欄代表每個可評分模組對整體課程分數的貢獻。

| **價值** | **描述** |
|----------------------------------------------|------------------------------------------------------|
| 數值百分比（例如20、30、50） | 模組對課程分數的貢獻 |
| 空白 | 模組不可評分（例如PDF或影片） |

### 內容審核報告

兩個新事件會反映成績簿設定的變更。

| **活動** | **觸發時間** | **擷取資料** |
|-----------------------|-----------------------------------------------------------------|----------------------------------------------------------|
| 成績單更新 | 成績簿在課程層級可啟用、停用或修改 | 成績簿狀態變更;評分配置更新 |
| 模組重量更新 | 模組的權重會被修改 | 模組識別碼;更新的權重值 |

學習者成績單反映最新的權重。 內容審核報告追蹤歷史變更。 這些元素合起來，讓你全面了解當前的計分邏輯及其演變。

## 外部學習報告變更

### 學習者成績單

新增三欄以支援外部學習記錄。

| **柱狀** | **描述** |
|------------------------|-----------------------------------------------------------------------------------------------------|
| 外部學習名稱 | 學習者提交的外部學習活動名稱 |
| 自訂欄位 | 每個自訂欄位可設定一欄供外部學習（文字、數字、勾選框或下拉選單） |
| 完成評論 | 經理在核准或拒絕時所輸入的意見 |

**注意：** 在學習者成績單（學習者自助檢視）中，欄位位置與行政學習者成績單不同：

- **外部學習名稱** 會立即在現有 **模組** 欄位後加入。

- **完成評論** 會緊接在現有 **的評論者評論** 欄之後。

- 自訂欄位欄位（每個配置欄位一欄）會附加在逐字稿末尾。

在行政學習者成績單中，所有新增欄位，包括外部學習名稱與完成評論，都會附加在末尾，接著是自訂欄位欄位。

### 學習者逐字稿中的類型欄位

外部學習項目現在會與現有的學習物件（課程、學習路徑、認證）一同出現在管理員 LT 中。 **類型**&#x200B;欄位包含新的外部學習分類，方便篩選。

外部學習資料會同時流入學習者成績單和行政LT。 核心欄位如完成日期、狀態及分數對應至現有欄位。 自訂欄位會作為額外欄位附加。

## 增量用戶回報變更

一個新的增量匯出模型允許你只匯出在指定時間內資料變動的使用者，而不必每次都產生完整的資料匯出。

| **匯出模式** | **行為** |
|--------------------|-----------------------------------------------------------------|
| 完整出口 | 回傳帳號中的所有使用者 |
| 增量出口 | 僅返回指定日期範圍內變更的使用者 |

要使用增量匯出，請依&#x200B;**日期與**&#x200B;**日期**&#x200B;篩選以定義變更視窗。使用者回報現在透過資料平台管道產生，輸出則以區塊形式回傳，以支援大型帳號。

## Gen AI 信用報告

一個新的信用儀表板和兩份報告，讓管理者能掌握生成式人工智慧的信用消費狀況。

### 信用儀表板

儀表板會在帳戶層級呈現以下指標。

| **公制** | **描述** |
|-------------------|---------------------------------------------------|
| 購買的信用點數 | 帳戶總額度 |
| 使用過的製作人員 | AI驅動功能中消費的點數 |
| 剩餘作品 | 消費後可用信用點數 |
| 各功能用途 | 信用消耗依個別 AI 功能分配 |

### 新報告

| **賽事報導** | **描述** |
|----------------------|---------------------------------------------------------------------------------------------|
| 每月使用報告 | 按月份、功能及消費信用點彙總信用消費 |
| 稽核追蹤報告 | 提供使用者層級細節：使用者識別碼、功能名稱、已使用點數及時間戳記 |

## 其他行為變化

### 根認證：根訓練識別碼

**在管理員學習者成績單**&#x200B;與&#x200B;**學習者成績單**（學習者自助檢視）的末尾&#x200B;**，新增了根訓練 ID** 欄位。它捕捉唯一識別碼，將所有認證的重複驗證連結到單一根實體。 這讓所有重複出現的認證實例都能與單一根 ID 關聯，以便追蹤與過濾。

### Webhook 與學習者文字稿時間戳標準化

Webhook 的時間戳現在已與學習者逐字稿格式對齊。 Webhook 有效載荷資料物件&#x200B;**中的**&#x200B;每個日期與時間欄位，秒數值皆設為 00，提供與學習者逐字稿報告一致的分鐘級細度。這消除了在比較 webhook 資料與 Learner Transcript 資料時，需要標準化時間戳格式。

### API 回應中的作者資訊，針對共享課程

當課程透過目錄共享從一個 Adobe Learning Manager 帳號分享到另一個帳號時，API 的學習物件（LO）回應現在只會回傳原始帳號的原始作者資料。 過去，接受帳號的管理員會在其帳號的 API 回應中以課程作者身份出現

此變更僅影響對等（接收）帳號。 當你查詢接收帳號中共享課程的 LO 詳細端點時，authorNames 欄位現在反映的是來源帳號的原始作者，而非接收帳號的管理員。

在來源帳號查詢 LO 時，作者資料的顯示方式沒有改變。

**注意：** 如果你的整合依賴於共享課程 LO API 回應中的 authorNames 欄位，請確認更新的作者資料沒有違反任何假設接收帳號管理員名稱會出現在此欄位的下游邏輯。
