# 疑難排解Salesforce (SFDC)與Adobe Learning Manager的整合問題

## 疑難排解SFDC匯出失敗（2-3小時以上無匯出）

如果SFDC匯出未成功完成超過&#x200B;**2-3小時**，請使用下列步驟來識別和瞭解問題。

1. 前往&#x200B;**Salesforce首頁**&#x200B;頁面。
2. 在&#x200B;**快速尋找**&#x200B;搜尋方塊中，輸入&#x200B;**`Bulk Data Load Jobs`**&#x200B;並開啟它。
3. 此頁面顯示&#x200B;**過去7天**&#x200B;的大量資料載入工作。
4. 尋找與&#x200B;**Adobe Learning Manager (ALM)物件**&#x200B;相關的工作。
5. 按一下您要調查的特定工作。
6. 捲動至工作詳細資料頁面的&#x200B;**底部**&#x200B;以檢視&#x200B;**錯誤訊息**&#x200B;和其他詳細資料。

在許多情況下，根本原因是SFDC **中的欄位層級許可權不足**。 失敗的欄位名稱可能如下所示：

- `cp_Module_ID`
- 或其他`cp_...`欄位與ALM自訂物件相關。

如果錯誤中出現此類欄位，請繼續下一節。

## 授予SFDC中ALM自訂物件欄位的欄位許可權

使用&#x200B;**欄位協助工具**&#x200B;功能修正特定欄位（例如`cp_Module_ID`）的許可權問題。

### 步驟

1. 前往&#x200B;**Salesforce首頁**&#x200B;頁面。
2. 在&#x200B;**快速尋找**&#x200B;搜尋方塊中，輸入&#x200B;**`Field Accessibility`**&#x200B;並開啟它。
3. 從物件清單中，選取相關的&#x200B;**報告型別/物件**。
   - 範例： `cp_LearnerTranscript_xxxx_xxxx`學習者成績單(LT)報表。
4. 在下一頁，選擇&#x200B;**「依欄位檢視」**。
5. 在&#x200B;**欄位下拉式清單**&#x200B;中，選取出現匯出錯誤的欄位
   - 範例： `cp_Module_ID`。
6. 在存取矩陣中，按一下&#x200B;**系統管理員**&#x200B;設定檔的&#x200B;**「隱藏」**&#x200B;專案（如有需要，可按一下其他相關設定檔）。
7. 在下一頁：
   - 在適當的時候啟用&#x200B;**「可見」**。
   - 按一下頁面底部的&#x200B;**儲存**。
8. 儲存後，您會返回欄位協助工具檢視。 對於&#x200B;**系統管理員** （以及您更新的任何其他設定檔），欄位現在應該顯示為&#x200B;**「可編輯」**。

## 對所有受影響的欄位重複上述步驟，然後再次嘗試匯出

1. **重複**&#x200B;第2節中針對&#x200B;**每個欄位**&#x200B;報告的欄位協助工具步驟，在&#x200B;**大量資料載入工作**&#x200B;中有許可權問題。
2. 一旦所有有問題的欄位都具備適當的可見度和編輯許可權：
   - 返回&#x200B;**Adobe Learning Manager**。
   - 在&#x200B;**SFDC聯結器**&#x200B;設定中，**重試匯出**。


## 重要附註和限制

在設計整合或疑難排解整合時，請謹記這些SFDC-ALM細節。

### SFDC中不會自動建立物件/欄位

- **SFDC聯結器沒有在Salesforce**&#x200B;中建立新物件或欄位。
- 如果在ALM **中新增**&#x200B;新欄位，而您希望它出現在SFDC中：
   - 在SFDC中手動&#x200B;**建立對應的自訂欄位**。
   - **將** SFDC自訂欄位對應到聯結器設定中的&#x200B;**適當的ALM欄位**。
   - 確定新欄位具有&#x200B;**適當的欄位層級許可權** （使用區段2）。

### 具有自訂網域的ALM帳戶回呼URL

- 如果ALM帳戶使用&#x200B;**自訂網域**，工程團隊必須將該自訂網域&#x200B;**新增至SFDC聯結器的OAuth生產應用程式的**&#x200B;回呼URL允許清單&#x200B;**。**
- 這是透過對工程的&#x200B;**Jira要求**&#x200B;完成的。
- 若無此專案，聯結器的OAuth流程可能會失敗。

### 時區限制（僅限UTC）

- 如果SFDC組織使用UTC **以外的**&#x200B;時區，同步期間可能會遺失&#x200B;**完成和註冊資料**。
- 原因： ALM **目前僅支援SFDC整合的UTC時區**。
- 內部參考：引發PAPI-24525以支援其他時區。

### 篩選器資料型別限制（挑選清單與檢查清單）

- ALM支援匯入使用挑選清單欄位&#x200B;**建立的** SFDC篩選器。
- ALM **不支援根據檢查清單/多重選取核取方塊欄位的**&#x200B;篩選器。
- 原因： ALM **僅支援SFDC篩選的字串資料型別**。
